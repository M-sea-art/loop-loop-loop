# LoopLoopLoop v2 — Autonomous Goal Completion Engine

## Core Purpose

LoopLoopLoop is not a virtual company simulator. It is an autonomous goal completion engine.

The user's basic need is:

> AI should understand a desired outcome, define a reasonable goal, execute continuously, verify reality, and improve until the goal reaches the ideal state.

## Core Loop

```
Intent
  ↓
Goal Architect
  ↓
Plan
  ↓
Execute
  ↓
Verify Reality
  ↓
Find Gap
  ↓
Improve
  ↓
Verified Completion
```

## Minimal Roles

Keep the system intentionally small:

- Goal Architect: defines desired state and acceptance criteria.
- Planner: creates execution path.
- Worker: performs changes.
- Judge: independently verifies reality.

New roles must prove they improve completion reliability.

## Non-expansion Rules

LoopLoopLoop must not grow by adding concepts, agents, or processes without evidence.

Every addition must improve at least one:

1. Goal completion rate.
2. Verification reliability.
3. Reduction of human intervention.

## Completion Principle

Activity is not completion.

Code written, reports generated, or agents claiming success are only signals.

Completion requires:

- frozen goal contract;
- direct evidence;
- independent verification;
- acceptance gate approval.

## Future Direction

LoopLoopLoop should remain a focused runtime for turning human intent into verified outcomes.
