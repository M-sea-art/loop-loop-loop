# LoopLoopLoop v2 — Autonomous Goal Completion Engine

## Core Positioning

LoopLoopLoop is not an AI company simulator and not a collection of prompts.

It is an autonomous goal completion engine:

> Given a human intention, AI defines the desired state, builds a path, executes work, verifies reality, and continues improving until the goal is achieved.

## Core Flow

```
User Intent
    ↓
Goal Architect
    ↓
Execution Plan
    ↓
Worker Loop
    ↓
Evidence Collection
    ↓
Independent Judge
    ↓
Improvement Loop
    ↓
Verified Goal Completion
```

## Non-Expansion Rules

LoopLoopLoop must optimize for goal completion, not system complexity.

Do not add:

- unnecessary agents
- simulated company structures
- management layers
- workflows without measurable improvement

Any new capability must prove at least one:

1. increases goal completion probability;
2. reduces failure rate;
3. reduces human intervention.

## Minimal Runtime Model

The preferred default model:

- Goal: what outcome is required
- Planner: how to reach it
- Executor: performs actions
- Judge: independently verifies reality
- State: preserves progress and evidence

## Completion Principle

Task completion is not goal completion.

A goal is complete only when:

- desired state is defined;
- acceptance conditions are satisfied;
- evidence exists;
- independent verification agrees;
- remaining gap is acceptable.

## Future Direction

LoopLoopLoop should become an operating layer for AI systems that are responsible for outcomes, not merely responses.
